2026.4.26.11: move Map tab from switch topology to per-agent network graph

The previous build added a "Map" tab to the switch fabric topology
(/topology) — that was the wrong target. The Map view belongs on the
per-agent Network Topology card (the radial host→process→ISP graph)
because that's where remote peers with public-IP GeoIP fixes live.

This commit:
- AgentNetworkGraph now has Graph / Map tabs that share the same
  Direction / Scope / Process filters. The Map plots remote peers on
  a zoomable react-simple-maps world map, draws great-circle lines
  from the agent to each peer (when the agent has its own GeoIP),
  colours by direction (outbound = sky blue, inbound = amber),
  clusters nearby peers at low zoom, and surfaces unmappable peers
  (private networks, anycast) in a side panel. Tab choice persists
  to localStorage.
- Topology page reverts to graph-only — the tabs, TopologyMap, and
  UnmappedPanel are removed. The TagFilter dropdown stays.
- handlers_topology.go drops the per-node MaxMind enrichment and
  IsPrivate / lat / lon / asn / org fields that were only there to
  feed the now-removed Topology Map. Tags stay (still used by the
  TagFilter dropdown).
- web/src/api/types.ts mirrors the backend by removing the geo
  fields from TopologyNode.
