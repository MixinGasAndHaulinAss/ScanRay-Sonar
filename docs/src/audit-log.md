# Audit log

**Audit** is a **superadmin** security trail of sensitive actions (user changes, credential updates, and similar events depending on what the API records).

## How to use it

1. Open **Audit**.
2. Filter by time or actor when controls are present.
3. Investigate unexpected changes before rotating credentials or revoking access.

Pair audit review with [Users](users.md) and [API keys](api-keys.md) when responding to an incident.
